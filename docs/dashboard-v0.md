# NetSentry Dashboard V0

## Goal

Create a small local web dashboard that displays Snort IDS alerts for the NetSentry V0 lab.

The dashboard is not public. It is intended to be accessed only from the admin laptop.

## Current Role

NetSentry is still in V0 host-facing mode.

That means:

```text
Snort detects traffic hitting the Debian NetSentry machine itself.
NetSentry is not yet a full gateway inspecting all LAN-to-WAN traffic.
```

## Dashboard Script

```text
scripts/netsentry_dashboard.py
```

## Alert Input File

The dashboard reads structured alerts from:

```text
~/netsentry-gateway/snort/alerts/alerts.jsonl
```

This file is generated by:

```text
scripts/snort_alert_watcher.py
```

## Raw Snort Alert File

Raw Snort alerts are stored in:

```text
~/netsentry-gateway/snort/alerts/alert_fast.txt
```

Snort writes to it using:

```bash
cd ~/netsentry-gateway

sudo stdbuf -oL -eL snort -c /usr/local/etc/snort/snort.lua \
-R ~/netsentry-gateway/snort/rules/local.rules \
-i wlx200db0220b9a \
-A alert_fast | stdbuf -oL tee -a ~/netsentry-gateway/snort/alerts/alert_fast.txt
```

The `stdbuf` part is important because it prevents partial-line buffering problems.

## Run Commands

Use three terminals.

### Terminal 1 — Snort

```bash
cd ~/netsentry-gateway

sudo stdbuf -oL -eL snort -c /usr/local/etc/snort/snort.lua \
-R ~/netsentry-gateway/snort/rules/local.rules \
-i wlx200db0220b9a \
-A alert_fast | stdbuf -oL tee -a ~/netsentry-gateway/snort/alerts/alert_fast.txt
```

### Terminal 2 — Alert Watcher

```bash
cd ~/netsentry-gateway
python3 scripts/snort_alert_watcher.py
```

### Terminal 3 — Dashboard

```bash
cd ~/netsentry-gateway
python3 scripts/netsentry_dashboard.py
```

## URL

From the admin laptop:

```text
http://192.168.1.17:5050
```

## Dashboard V0 Features

- Shows latest Snort alerts
- Shows total alerts loaded
- Shows matching alerts after filters
- Shows latest source IP
- Shows latest destination port
- Shows latest SID
- Shows source IP and source port
- Shows destination IP and destination port
- Shows protocol
- Shows SID and revision
- Shows priority
- Shows raw alert line
- Auto-refreshes every 10 seconds

## Filters

The dashboard supports filters by:

- Source IP
- Destination IP
- Destination port
- Protocol
- SID
- Revision
- Message text

Example filtered URLs:

```text
http://192.168.1.17:5050?dst_port=22
http://192.168.1.17:5050?sid=10000002
http://192.168.1.17:5050?rev=2
http://192.168.1.17:5050?src=192.168.1.30
http://192.168.1.17:5050?dst_port=5050&q=Dashboard
```

## Alert Summary Cards

The dashboard includes summary cards for known SIDs:

| SID | Meaning |
|---|---|
| `10000001` | ICMP Ping |
| `10000002` | SSH SYN Probe |
| `10000003` | SSH Brute-force / Scan |
| `10000004` | AdGuard UI Attempt |
| `10000005` | DNS Query |
| `10000006` | Dashboard Access Attempt |

These cards show how many alerts of each type are currently loaded.

## Reset Button

The dashboard includes a reset button.

It clears:

```text
snort/alerts/alert_fast.txt
snort/alerts/alerts.jsonl
```

This makes the dashboard empty again.

Best clean reset process:

```text
1. Stop Snort.
2. Stop the watcher.
3. Click reset.
4. Start Snort again.
5. Start watcher again.
```

If Snort and the watcher are still running during reset, new alerts may appear again immediately. That is normal.

## API JSON Endpoint

The dashboard exposes an API endpoint:

```text
http://192.168.1.17:5050/api/alerts
```

This returns the alerts as JSON.

The API supports the same filters as the web dashboard.

Examples:

```text
http://192.168.1.17:5050/api/alerts?dst_port=22
http://192.168.1.17:5050/api/alerts?sid=10000002
http://192.168.1.17:5050/api/alerts?src=192.168.1.30
```

Purpose:

```text
The HTML dashboard is for humans.
The API is for scripts, future live-refresh, mobile tools, or ESP32 integrations.
```

## Firewall Protection

The dashboard is protected by iptables.

Policy:

```text
Allow dashboard access from admin laptop only: 192.168.1.11
Block dashboard access from everyone else.
```

Rules added:

```bash
sudo iptables -I INPUT 7 -p tcp -s 192.168.1.11 --dport 5050 -j ACCEPT
sudo iptables -I INPUT 8 -p tcp --dport 5050 -j DROP
```

## Snort Protection

Snort detects dashboard access attempts from non-admin clients.

Rule:

```snort
alert tcp !192.168.1.11 any -> 192.168.1.17 5050 (
msg:"NETSENTRY Dashboard access attempt from non-admin detected";
flags:S;
sid:10000006;
rev:1;
)
```
