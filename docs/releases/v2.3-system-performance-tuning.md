## Hardware Constraint

NetSentry runs on a low-resource Debian gateway.

Because Wazuh and Suricata are heavier security components, the system needs conservative tuning.

---

## Applied Changes

### Docker Disabled

Docker was not required for the active NetSentry gateway stack, so the following services were disabled:

```text
docker.service
docker.socket
containerd.service
```

This reduces unnecessary background services at boot.

### Wazuh Indexer Heap Reduced

Wazuh indexer heap is tuned to:

```text
-Xms512m
-Xmx512m
```

This keeps Wazuh usable without consuming the default larger heap.

### Wazuh Vulnerability Detection Disabled

Wazuh vulnerability detection is disabled for this homelab gateway:

```xml
<vulnerability-detection>
  <enabled>no</enabled>
</vulnerability-detection>
```

This reduces CPU and memory pressure.

### Wazuh Syscheck Tuned

File integrity monitoring remains enabled, but is made lighter:

```text
frequency = 86400
scan_on_start = no
```

This avoids expensive scans every time Wazuh manager restarts.

### Swap Safety Buffer

Disk-backed swap is used as a safety buffer.

Swap is not real RAM. It prevents crashes during memory pressure, but heavy swap usage can make an HDD-based system lag.

Target:

```text
swap = around 6 GB
swappiness = 10
```

### Services Kept Enabled

The following services remain part of the active NetSentry stack:

```text
ssh
nginx
netsentry-web
netsentry-ap-interface
netsentry-firewall
netsentry-dnsmasq
hostapd
AdGuardHome
tailscaled
suricata
wazuh-indexer
wazuh-manager
wazuh-dashboard
filebeat
getty@tty1
```

`getty@tty1` is intentionally kept enabled because it provides emergency local console access.

## Expected Result

At idle, the system should remain usable with:

- Low CPU usage
- Acceptable RAM pressure
- Swap available as emergency buffer
- Wazuh dashboard usable
- NetSentry dashboard responsive
- Suricata still running
- No Docker/containerd boot overhead

## Success Criteria

This branch is successful when:

- Docker and containerd are disabled if unused
- Wazuh heap remains 512m
- Vulnerability detection remains disabled
- Syscheck does not scan on start
- Swap is available
- NetSentry dashboard works
- Wazuh dashboard works
- Suricata works
- System remains responsive after reboot
