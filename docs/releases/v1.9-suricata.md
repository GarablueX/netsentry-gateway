# NetSentry v1.9 — Suricata IDS Layer

## Goal

NetSentry v1.9 adds Suricata as a modern IDS layer beside the existing Snort AP-side IDS.

The goal is not to immediately replace Snort. The goal is to run Suricata side-by-side, use its structured `eve.json` output, and compare Suricata detection with the current Snort-based IDS pipeline.

## Why Suricata

Suricata gives NetSentry:

- Modern IDS/IPS engine
- Native JSON event output through `eve.json`
- Better structured alert parsing
- Strong SOC/blue-team relevance
- Cleaner dashboard integration
- A path toward multi-engine detection
