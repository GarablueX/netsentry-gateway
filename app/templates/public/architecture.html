Public Side

/, /about, /features, /architecture, /status, /docs, /hardware, and /contact explain the project without exposing private logs or client data.

Admin Side

/admin/* shows dashboard, clients, DNS, firewall, IDS, logs, and network state. It requires login and is also protected by Nginx LAN-only access.

API Side

/api/* returns structured read-only data from Linux commands, AdGuard, Snort alert files, DHCP leases, and logs.

Data Sources

Security Boundary

Flask stays localhost-only. Nginx is the front door. Public routes show safe project information only. Admin/API routes are protected by login and LAN-only Nginx rules. Active firewall changes, packet captures, and service restarts are postponed until a privileged agent exists.
